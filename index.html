How Web Browsers Work

A web browser’s primary job is to turn code into a visual, interactive experience. This happens in three main phases:

1. The Request (Fetching)

  • DNS Lookup: The browser finds the "IP address" (server location) for the domain name you typed.
  • HTTP Request: It sends a formal request to that server for the site's files (HTML, CSS, JS).
  • Response: The server sends the files back in small data packets.

2. The Parsing (Organizing)

  • DOM Building: The browser reads HTML to create a map of the content (text, buttons, headers).
  • CSSOM Building: It reads CSS to understand the "look" (colors, fonts, layout).
  • JavaScript: It executes code to make the page interactive (menus, animations).

3. The Rendering (Displaying)

  • Render Tree: The browser combines content (DOM) and styles (CSSOM) into a final blueprint.
  • Layout: It calculates exactly where every element fits based on your screen size.
  • Painting & Compositing: It pixels in the colors and layers the elements to show the final image.

Core Components

  • User Interface: The address bar, buttons, and tabs you interact with.
  • Rendering Engine: The "brain" that draws the page (e.g., Blink, Gecko).
  • Browser Cache: Stores files locally so repeat visits load instantly.

The Internet: How it Works & Who Regulates It

How it Works

  • Physical Infrastructure: Data travels via undersea fiber-optic cables, satellites, and routers provided by ISPs.
  • Packet Switching: Information is sliced into tiny packets that travel independently and reassemble at the destination.
  • TCP/IP Protocols: IP handles the addressing (where it goes), while TCP ensures the data is complete and in order.

Who Regulates It?

The internet uses a multistakeholder model rather than a single ruler:

Organization Primary Role
ICANN Manages the "Address Book" (IP addresses and Domain Names).
IETF Sets the technical "Rules" (Protocols) like TCP/IP.
W3C Maintains Web Standards (HTML, CSS).
Governments Enforce local laws (Privacy, Security, and Content) within borders.

Static vs. Dynamic Web Pages

Static Web Pages

Files are stored on the server exactly as the user receives them. Content remains the same for every visitor until the developer manually changes the source code.

Common Tech: HTML CSS JS

Dynamic Web Pages

Content is generated in real-time. The server retrieves data from a database and constructs the page uniquely for each user (e.g., login screens, dashboards).

Common Tech: PHP Python MySQL

Feature Static Dynamic
Content Fixed; same for everyone. Personalized; changes per user.
Complexity Low; simple to build and host. High; requires backend logic.
Database No database required. Connects to a database.
Loading Speed Very Fast (no server processing). Slower (server must "build" page).
Security High (fewer entry points). Moderate (vulnerable to SQL attacks).

Web 3.0 Explained

Core Points of Web 3.0

  • Ownership: Users own their data and digital assets (via tokens/NFTs) rather than big tech companies.
  • Decentralization: No central server; data is stored across a global network of computers (Blockchain).
  • Permissionless: Anyone can use the network without needing approval from a governing body or company.
  • Semantic Web: AI helps computers understand the "meaning" of data, not just keywords, for smarter results.
  • Trustless: You don't need to "trust" a third party (like a bank) because transactions are verified by code (Smart Contracts).
